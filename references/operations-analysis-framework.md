# 电商运营竞品分析框架

本文件用于 Step 8/Step 9 的策略分析，也用于用户提供本地 JSON 时直接生成竞品分析报告。当前 skill 定位为灯具/照明行业决策助手，默认目标是提升转化，默认输出为单一主决策，而不是多方案平铺。

## 1. 输入识别

优先识别：

- 自家商品文件、商品 ID 或商品链接。
- 竞品文件、商品 ID 或商品链接。
- 平台、类目、商品标题、价格、销量/评价数、店铺名称、店铺评分。
- 商品主图、详情页、评价内容、问答内容。

常见 JSON 字段：

- `商品ID`
- `商品标题`
- `店铺名称`
- `价格.原价`
- `价格.券后价`
- `已售数量`
- `评价总数`
- `店铺评分.宝贝质量`
- `店铺评分.服务保障`
- `店铺评分.物流速度`
- `商品主图`
- `详情图`
- `feedback`
- `question`

Windows 下读取中文 JSON 时使用 UTF-8，例如 `Get-Content -Encoding UTF8`。

## 2. 必做分析维度

### 决策机制

Step 9 必须先判断“能不能决策”，再判断“做什么决策”。

`decision_type` 固定为：

- `execute`：数据足够，且多个证据指向同一阻力，可以直接执行。
- `experiment`：判断方向明确，但缺真实转化数据，需要 A/B 测试。
- `collect_data`：缺少关键数据，先补评价、问答、主图、SKU 或点击率。
- `hold`：当前动作可能造成误改或浪费，本轮不建议行动。

默认转化优先级为：

`标题搜索承接 > 首图点击理由 > SKU选择成本 > 详情页信任链 > 评价/问答补强 > 投放放大`

最终输出必须包含一个主决策、Top3 动作、本轮不做事项、置信度和验证方案。

### 数据质量门槛

缺失数据时，必须限制对应结论：

| 缺失数据 | 决策限制 |
|---|---|
| 缺标题 | 不能判断搜索承接 |
| 缺主图 | 不能判断点击理由 |
| 缺 SKU/规格 | 不能判断选择成本 |
| 缺评价/问答 | 不能判断用户焦虑 |
| 缺价格 | 不能判断价格阻力 |
| 缺竞品 | 不能做竞争策略，只能做单品诊断 |

置信度判断：

- 高：价格、标题、SKU、评价、问答、主图、竞品数据都较完整，且多个证据指向同一问题。
- 中：关键数据基本完整，但视觉、详情页、评价或转化数据中有部分缺失。
- 低：缺少评价、问答、图片、SKU 或竞品样本，无法确认真实转化阻力。

### 证据链

关键判断必须形成：

`原始数据 → 运营解释 → 决策判断 → 执行动作`

禁止只写“建议优化标题/主图”。必须说明为什么这个动作优先于其他动作。

### 转化阻力诊断

每个阻力至少包含：

- 阻力名称。
- 严重程度：high / medium / low。
- 证据。
- 影响阶段：搜索点击前、点击决策、加购决策、下单决策、流量放大。
- 优先级：P0 / P1 / P2。

最高优先级阻力必须推导出最终主决策。

### 灯具行业决策规则库

1. 如果标题缺少“吸顶灯/客厅主灯/全光谱/护眼/智能/适用面积”等词，而 SKU、评价或问答中存在这些卖点，优先判断为“搜索承接弱”。
2. 如果评价/问答高频出现“够不够亮、多少平、刺不刺眼、安装费”，详情页必须优先补“面积选型 + 亮度说明 + 安装说明”。
3. 如果商品价格明显低于品牌竞品，但评价体量弱，不能主打低价，应主打“同配置更省 + 服务保障”。
4. 如果出现“塑料感、廉价、色差、不开灯不好看”等疑虑，主图/详情页必须补材质近景和不开灯实拍。
5. 如果全屋套餐存在，但标题/首图没有表达，优先补套餐省钱心智，而不是只放在 SKU 里。
6. 如果声称护眼、无频闪、Ra97、全光谱，但缺少检测报告或证明素材，只能建议表达方向，并标注“需要证据素材支撑”。

### 基础对比矩阵

输出表格：

- 对象
- 商品定位
- 标价/到手价
- 销量/评价体量
- 店铺评分
- 核心卖点
- 主要优势
- 主要劣势/风险

必须量化价格差距和评价体量差距。

### 标题关键词拆解

拆分：

- 品牌词
- 类目词
- 功能词
- 场景词
- 风格词
- 价格/套餐词

必须指出：我方标题漏掉了哪些 SKU/评价/问答里反复出现的高转化词。

### 评价与问答洞察

分析原则：

- 好评代表成交理由。
- 问答代表下单阻力。
- 差评或负面表达代表突破口。
- 用户原话优先转化为主图、详情页和广告文案。

重点看：

- 质量
- 亮度/效果
- 是否适合具体场景
- 安装和售后
- 正品/品牌信任
- 廉价感/质感
- 耐用性
- 价格保护/退差价

### 价格与 SKU

必须判断：

- 引流款
- 主推款
- 利润款
- 升级款
- 套餐款

SKU 命名优先按用户决策方式重构：面积/尺寸、户型、功能、人群、使用场景、套餐省钱。

### 竞品成功逻辑

逐个回答：

- 第一购买理由是什么。
- 降低了哪类风险。
- 标题、主图、详情页、SKU、评价、问答如何共同促进转化。
- 是靠品牌、价格、功能、套餐、服务、视觉、内容还是渠道成交。

## 3. 标准报告结构

每次输出固定使用：

1. 一、竞品分析总览
2. 二、核心竞争结论
3. 三、竞品优势拆解
4. 四、竞品弱点与突破口
5. 五、我方优化策略
6. 六、30天行动计划
7. 七、最终建议

## 4. 我方优化策略必含内容

### 本轮主决策

必须输出唯一主决策，格式为：

- 本轮主决策。
- 为什么是这个。
- 为什么不是其他。
- Top3 动作。
- 本轮暂不做什么。
- 最大风险。
- 置信度。

不允许把 Plan A/B/C 平行展示成“都可以做”。Plan A/B/C 只能作为支撑材料。

### 标题优化

给出至少 2 个可测试标题。

标题结构建议：

`品牌 + 核心品类 + 核心功能 + 场景 + 年份/风格 + 智能/服务/套餐`

### 主图优化

逐张说明任务：

1. 第一张：首屏购买理由。
2. 第二张：核心痛点证明。
3. 第三张：功能/参数证明。
4. 第四张：材质/细节/质感证明。
5. 第五张：套餐/价格/服务证明。

### 详情页优化

按用户决策顺序重排：

1. 适合谁。
2. 解决什么痛点。
3. 为什么可信。
4. 怎么选 SKU。
5. 为什么比竞品值。
6. 安装/售后怎么保障。
7. 用户评价怎么证明。

### 价格策略

设计价格梯度：

- 引流款
- 主推款
- 利润款
- 套餐款

### 评价引导

评价引导模板必须包含：

- 使用场景/面积/户型
- 购买 SKU
- 核心体验
- 是否解决痛点
- 安装/客服/售后体验
- 实拍图建议

### 投放与内容

输出：

- 搜索词
- 竞品词
- 场景词
- 痛点词
- 短视频/直播/达人内容主题

## 5. 30天行动计划

建议结构：

- 第1周：修标题、首图、SKU命名、选择指南。
- 第2周：重做详情页信任链和服务证明。
- 第3周：测试广告词、价格梯度、内容主题和竞品拦截。
- 第4周：放大高转化 SKU/素材，沉淀评价和问答。

每周必须包含验收标准。没有真实 baseline 时，写“新旧版本对比胜出”，不要写“提升20%”。

## 6. 质量规则

- 每条重要判断都要能回到字段、评价、问答、标题/SKU 或页面元素。
- 不只写“优化标题/主图/详情页”，必须写清楚怎么改。
- 不只描述竞品卖点，要指出背后的商业逻辑。
- 区分“便宜”和“高性价比”，区分“贵”和“信任溢价”。
- 输出要能让运营、美工、投放、客服直接执行。
